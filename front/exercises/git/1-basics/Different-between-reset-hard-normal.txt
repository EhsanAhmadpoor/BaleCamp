در حالت ریست هارد تمامی تغییرات در ورکینگ دایرکتوری از بین میرود. همچنین تغییرات که استیج شده اند از حالت استیج خارج میشوند تا به حالت کامیت برگردد. همچنین هیستوری برنچ به یک کامیت خاص بازمیگردد و تمامی کامیت های بعد از کامیت مشخص شده از هستوری برنچ حذف میشوند. زمانی که میخواهیم همه ی تغییرات را از دست بدهیم و دوباره از یک کامیت خاص شروع به کار کنیم استفاده از این ریست مناسب است.
در حالت ریست معمولی ( یا همون ریست میکسد) ورکینگ دایرکتوری بدون تغییر میماند. اما تغییرات استیج شده از حالت استیج خارج میشود تا به وضعیت کامیت بازگردد. هیستوری برنچ به کامیت مشخص شده باز میگردد. استفاده از این نوع ریست زمانی مناسب است که میخواهیم تغییرات استیج شده برگردد اما تغییرات ورکینگ دایرکتوری از دست نرود.
کامند های مربوط به هرکدام:
git reset <commit>
یا
git reset --mixed <commit>

git reset --hard <commit>
